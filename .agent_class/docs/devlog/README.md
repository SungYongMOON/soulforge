# .agent_class/docs/devlog

## 목적

- `devlog/` 는 class 계층 작업의 실제 수행 결과를 기록한다.
- 계획이 아니라 수행 후 기록을 모은다.

## 포함 대상

- 날짜별 작업 로그
- 변경 이유, 영향, 남은 리스크
- `YYYY-MM-DD_<topic>_log.md` 형식의 기록 파일

## 제외 대상

- 수행 전 계획
- 저장소 전체 구조 문서와 프로젝트 전용 로그

## 관련 경로

- [`.agent_class/docs/README.md`](../README.md)
- [`.agent_class/docs/plans/README.md`](../plans/README.md)

## 상태

- Stable
- class 작업 단위별 로그를 유지한다. project 전용 로그는 각 프로젝트의 `.project_agent/` 아래에 둔다.
