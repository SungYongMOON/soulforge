Party Naming Draft v0

이 HTML은 사람이 빠르게 검토하기 위한 파생 정적 문서입니다. 정본은 Markdown/YAML/JSON 이며, 이 문서는 rename, alias catalog 등록, validator enforcement, approval 확정을 만들지 않습니다.

Status: draft Owner: .party Claim ceiling: canon_candidate

2026-05-18 기준 관찰된 `.party/index.yaml` 등록 party 3개의 한국어 alias 후보와 설명용 표시 이름을 한 화면에서 검토하기 위한 파생 뷰입니다.

빠른 요약

  • 기존 canonical party_id, folder, index 값은 유지한다.
  • global_name_ko 는 slash 없는 한국어 party alias 후보일 뿐이며 아직 catalog 에 등록되지 않았다.
  • display_name_ko 는 README, 작전판, UI 에서 쓰는 설명용 이름이며 호출 key 가 아니다.
  • party 는 reusable composition template 이며 workflow body, project-local run truth, raw battle log 를 소유하지 않는다.

이름 3계층 요약

표시 이름 display_name_ko 사람이 party 의 조합 성격을 읽기 위한 설명용 이름입니다. 문서용 slash 표기는 허용하지만 호출 key 로 쓰지 않습니다.
한글 전역 호출명 global_name_ko slash, 공백, dash, colon 없는 한국어 underscore alias 후보입니다. 한 alias 는 하나의 party 로만 resolve 되어야 합니다.
canonical party_id `.party/index.yaml`, party package path, workflow/mission resolve 가 쓰는 내부 안정 id 입니다.

금지 패턴 / 주의점

  • template, lineup pattern, role composition 같은 설명어를 실제 canonical party_id 와 섞지 않는다.
  • 특정 run 성과, mission success, owner acceptance 를 party 이름에 넣지 않는다.
  • 기존 party_id 를 미관이나 표시 언어 선호만으로 rename 하지 않는다.
  • party 이름이 workflow 실행 결과나 project performance 를 claim 하게 만들지 않는다.
  • workflow alias 와 party alias 를 같은 namespace 에 둘 경우 충돌 방지 정책이 먼저 필요하다.

Resolve 흐름

문서상 권장 흐름은 아래와 같습니다. 이 HTML은 해당 흐름을 설명만 하며 새 실행 surface 를 만들지 않습니다.

global_name_ko -> workflow_id -> party_id -> path

workflow alias 는 먼저 canonical workflow_id 로 resolve 되고, party 선택은 default_workflow_id, allowed_workflows.yaml, 요청 context, owner policy 를 함께 본 뒤 canonical party_id 로 resolve 되어야 합니다.
