2026-05-18 기준 관찰된 `.workflow/index.yaml` 등록 workflow 44개의 한국어 alias 후보와 설명용 표시 이름을 한 화면에서 검토하기 위한 파생 뷰입니다.

빠른 요약

  • 기존 canonical workflow_id, folder, index 값은 유지한다.
  • global_name_ko 는 slash 없는 한국어 호출 alias 후보일 뿐이며 아직 catalog 에 등록되지 않았다.
  • display_name_ko 는 README, UI, authoring note 에서 쓰는 설명용 이름이며 호출 key 가 아니다.
  • 실행 또는 저장 경계에서는 사람이 부른 이름을 반드시 canonical workflow_id 로 resolve 해야 한다.

이름 3계층 요약

표시 이름 display_name_ko 사람이 읽는 설명용 이름입니다. 문서용 slash 표기는 허용하지만 호출 key 로 쓰지 않습니다.
한글 전역 호출명 global_name_ko slash, 공백, dash, colon 없는 한국어 underscore alias 후보입니다. 한 alias 는 하나의 workflow 로만 resolve 되어야 합니다.
canonical workflow_id `.workflow/index.yaml`, workflow package path, runner, downstream ref 가 쓰는 내부 안정 id 입니다.

금지 패턴 / 주의점

  • 세계관 은유만 있고 실제 대상/행위가 없는 이름을 새 호출명처럼 쓰지 않는다.
  • approve, certify, accept, run, verify 같은 강한 claim 은 실제 권한과 근거 없이 쓰지 않는다.
  • private/raw/source payload 를 public canon 으로 끌어오는 듯한 이름을 피한다.
  • 기존 downstream ref 가 있는 workflow_id 를 미관만으로 rename 하지 않는다.
  • global_name_ko, 표시 이름, workflow_id 가 서로 다른 owner 나 권한 한계를 암시하지 않게 한다.

Resolve 흐름

문서상 권장 흐름은 아래와 같습니다. 이 HTML은 해당 흐름을 설명만 하며 새 실행 surface 를 만들지 않습니다.

workflow global_name_ko 후보 -> workflow_id -> party_id -> path

workflow alias 후보는 먼저 `.workflow/index.yaml` 의 canonical workflow_id 로 resolve 되고, party 선택은 `.party/index.yaml` 의 canonical party_id -> path catalog 로 이어져야 합니다.
