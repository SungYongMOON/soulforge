# 팀 공통 실험 보고서 목차 Reference

이 파일은 실험/시험/기술검토 보고서 작성 시 가져다 쓰는 참고 목차다.

보고서 성격에 따라 세부 항목은 조정할 수 있지만, 결과 보고서의 기본 흐름은 아래 순서를 유지한다.

```text
문서 개요 -> 시험 목적 -> 시험 대상 및 조건 -> 취득 데이터 -> 요청 및 검토 항목
-> 분석 방법 -> 시험 결과 -> 결과 해석 -> 고려사항 및 한계
-> 종합 판단 -> 후속 조치 -> 문서 관리
```

## 작성 주의

- HTML 목차를 만들 때는 아래 제목 번호를 그대로 사용한다. 목차 목록 자체에는 자동 번호를 붙이지 않는다.
- 검토 범위와 판정 한계는 보고서 첫머리의 `자료 성격`, `검토 범위`, `본 자료만으로 판단하지 않는 항목`에 자연스럽게 적는다.
- `이 문서는 ... 아닙니다`처럼 AI가 붙인 면책 문구로 보이는 배너 문장은 피한다.
- 계약상 수락 판정, 고객 승인, 최종 합격 기준 적용 여부는 별도 확인 대상으로 쓴다.

## 1. 문서 개요

- 과제명
- 시험명
- 시험일자
- 작성일자
- 작성자
- 핵심 요약
  - 검토 대상을 한 문장으로 제시
  - `검토 항목 / 결과 요약` 표로 판단, 핵심 수치, 해석 한계 정리
  - 파일명, 채널명, 계산 절차는 필요할 때만 근거로 언급
  - 거리 환산값은 확정 거리로 단정하지 않고 시험 배치 검토용 값인지 구분
- 관련 자료
- 판정 범위

## 2. 시험 목적

- 본 시험을 수행한 이유
- 확인하려는 기술 항목
- 본 시험 결과가 사용될 범위
- 본 시험으로 판단하지 않는 항목
