# intake_inbox

- `intake_inbox/` 는 mail 1건당 inbox container 1개가 생기는 sample 을 보여준다.
- `hiworks_dedupe_demo_001/` 는 새 monster 생성 예시다.
- `hiworks_dedupe_demo_002/` 는 기존 monster 연결 예시다.
- `pdr_brief_chain_demo_001/` 는 owner-facing brief 와 project-side battle/morning chain 까지 이어지는 예시다.
- 같은 실제 일은 `dedupe_key` 가 맞으면 새 monster 를 만들지 않고 기존 monster 를 갱신한다.
