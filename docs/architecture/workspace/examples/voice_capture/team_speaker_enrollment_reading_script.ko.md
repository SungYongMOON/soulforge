# 팀원 화자 등록 낭독문 v0

## 목적

이 문서는 실제 회의 공간과 PLAUD 배치 조건에서 팀원별 화자 등록 샘플을
일관되게 수집하기 위한 공통 낭독문이다. 등록 샘플은 화자 식별 후보를 만드는
로컬 파일럿 입력이며, 사람의 신원이나 회의 참석을 자동 확정하는 근거가 아니다.

## 녹음 전 준비

- 녹음과 로컬 화자 식별 사용에 동의한 팀원만 참여한다.
- 모든 팀원에게 `TEAM_01` 같은 가명 ID를 부여한다.
- 이름과 가명 ID의 연결표는 owner-approved 로컬 저장소에만 두며 Git이나
  `_workmeta`에 넣지 않는다.
- PLAUD를 실제 회의 때 사용할 위치에 두고, 다른 사람의 말·음악·키보드 소리가
  겹치지 않게 한 명씩 녹음한다.
- 연기하거나 과장하지 않고 평소 회의 음량과 말투로 읽는다.

## 공통 낭독문

```text
안녕하세요. 저는 [이름]이고, 담당 업무는 [담당 분야]입니다.

오늘 회의에서는 요구사항과 개발 일정, 시험 결과, 남아 있는 문제를 차례로
확인하겠습니다. 중요한 내용은 근거 자료와 함께 기록하고, 담당자와 완료 조건을
명확하게 정해야 합니다.

장비를 점검할 때는 전원과 연결 상태를 먼저 확인합니다. 센서, 케이블,
커패시터, 전류 프로브, 슬립링, 트랜스포머의 상태도 살펴봅니다. 측정값이 예상
범위를 벗어나면 검교정 이력과 시험 환경을 다시 확인합니다.

숫자와 영문 표기도 읽겠습니다. 하나, 둘, 셋, 열다섯, 이십칠, 백이십오.
에스이 오십, 아이피 육십칠, 사십 지, 씨피유, 아이피에스입니다.

일정이 변경되거나 새로운 위험이 발견되면 관련 팀에 공유하겠습니다. 지금 읽은
내용은 제 목소리를 구분하기 위한 등록용 문장입니다.

다시 한번 말씀드립니다. 제 이름은 [이름]입니다.
```

## 권장 수집 순서

1. PLAUD에서 약 1미터 거리로 공통 낭독문 전체를 한 번 읽는다.
2. 실제 좌석에 앉아 마지막 두 문단을 다시 읽는다.
3. 각 샘플의 가명 ID, 거리 조건, 녹음 시각, 기기 위치만 metadata로 기록한다.
4. 실제 녹음과 음성 특징은
   `_workspaces/system/voice_capture/speaker_enrollment/<speaker_id>/` 아래에만 둔다.
5. 첫 회의에서는 이름을 확정하지 않고 `UNKNOWN` 또는 화자 후보로만 비교한다.

## 완료 기준

- 참여자별 깨끗한 전체 낭독 1개와 실제 좌석 보조 낭독 1개가 있다.
- 서로 다른 팀원의 음성이 한 파일에 겹치지 않는다.
- 실제 이름·원본 음성·음성 특징이 public Git과 `_workmeta`에 없다.
- 낮은 신뢰도와 겹쳐 말한 구간은 `UNKNOWN`으로 유지한다.
