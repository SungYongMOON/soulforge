Soulforge Dev Worker Next Steps

2026-05-17 기준. 이 HTML은 사람 검토용 파생 산출물입니다. 정본은 Markdown/YAML/JSON 문서와 task packet입니다.

한 줄 결론

MacBook에서 후보 작업과 승인 규칙을 commit/push하면, 고성능 PC는 GitHub에서 최신 내용을 pull하고 승인된 ready task만 가져가 branch를 만들어 개발합니다. main merge는 자동화하지 않습니다.

구현됨: candidate queue 구현됨: approval gate 구현됨: auto-approval policy 구현됨: high-performance PC setup docs 다음: commit / push 다음: 고성능 PC pull / automation render

전체 흐름

1. 기억/비전 입력핸드폰이나 MacBook에서 제약, 비전, 해야 할 일을 말한다.
2. 후보 작성AI가 실행 전 후보를 dev_worker_candidate_queue에 쓴다.
3. 승인 판단owner 승인 또는 낮은 위험 auto-policy 승인만 통과한다.
4. 승격승격 helper가 ready task를 dev_worker_queue에 만든다.
5. 고성능 PC 개발worker가 ready task 하나를 claim하고 branch를 push한다.
6. 리뷰/머지MacBook 또는 supervisor가 branch를 검토하고 main merge를 결정한다.

지금 MacBook에서 할 일

  1. 현재 변경을 한 번 더 확인한다.
  2. commit을 만든다.
  3. public Soulforge를 push한다.
  4. 필요하면 companion repo인 _workmeta, private-state도 별도로 push한다.

고성능 PC 최초 세팅

고성능 PC에서 한 번만 실행합니다. Windows PowerShell이면 npm이 막힐 때 npm.cmd를 사용합니다.

git clone https://github.com/SungYongMOON/soulforge.git Soulforge
cd Soulforge
git clone https://github.com/SungYongMOON/soulforge-workmeta.git _workmeta
git clone https://github.com/SungYongMOON/soulforge-private-state.git private-state
npm install
npm install --prefix ui-workspace
npm run validate:dev-worker
npm run guild-hall:dev-worker:preflight -- --local-root "$PWD" --workmeta-root "$PWD/_workmeta" --private-state-root "$PWD/private-state" --json
npm run guild-hall:dev-worker:candidates -- --local-root "$PWD" --workmeta-root "$PWD/_workmeta" --json
npm run guild-hall:dev-worker:candidates -- --local-root "$PWD" --workmeta-root "$PWD/_workmeta" --auto-promote --json
npm run guild-hall:dev-worker:claim -- --local-root "$PWD" --workmeta-root "$PWD/_workmeta" --json

고성능 PC 자동화 생성

검증이 통과한 뒤 Codex local automation 파일을 생성합니다. 기본값은 PAUSED입니다.

npm run guild-hall:dev-worker:render -- --install --local-root "$PWD" --workmeta-root "$PWD/_workmeta" --private-state-root "$PWD/private-state"

생성 후 owner가 직접 활성화합니다. 활성화되면 기본 주기는 FREQ=HOURLY;INTERVAL=4이고, 한 번에 task 하나만 처리합니다.

작업 후보와 실행 작업의 차이

상태 위치 누가 씀 고성능 PC가 실행?
후보 _workmeta/<project>/dev_worker_candidate_queue/*.yaml AI, night_watch, owner 아니오
승인된 후보 dev_worker_candidate_queue 안에서 status: approved owner, supervisor, 또는 auto-policy helper 아직 아니오
실행 작업 _workmeta/<project>/dev_worker_queue/*.yaml 승격 helper 예
public-safe 실행 작업 .mission/<mission_id>/dev_worker_request.yaml MacBook/supervisor 예

후보 packet 예시

schema_version: soulforge.dev_worker_request.v0
task_id: improve_daily_packet_candidate
status: proposed
lane: dev_worker
requested_by: codex
project_code: system
summary: Add one missing field to the daily work packet report.
branch_slug: daily-packet-field
allowed_write_paths:
  - guild_hall/night_watch/**
  - guild_hall/dev_worker/**
acceptance_checks:
  - npm run validate:dev-worker
stop_conditions:
  - allowed_write_paths_insufficient
  - public_private_boundary_unclear
origin:
  kind: agent_generated
owner_approval:
  required: true
  approved: false
auto_approval:
  requested: true
  risk_level: low
notes:
  - Candidate only. Do not run on a worker until approved and promoted.

승인 또는 자동 승격

후보 파일에서 아래처럼 바꿉니다.

status: approved
owner_approval:
  required: true
  approved: true
  approved_by: owner
  approved_at: 2026-05-17T00:00:00.000Z

그다음 승격 helper를 실행합니다.

npm run guild-hall:dev-worker:candidates -- --promote-approved --json

낮은 위험 후보는 아래 명령으로 정책 평가, 승인, 승격을 한 번에 처리할 수 있습니다.

npm run guild-hall:dev-worker:candidates -- --auto-promote --json

그러면 _workmeta/<project>/dev_worker_queue/<task_id>.yaml에 ready task가 만들어집니다.

일일 보고서에서 볼 것

  • 새로 발견된 candidate 수
  • 승격 가능한 candidate 수
  • 자동 승인 가능한 candidate 수
  • 현재 ready task가 있는지
  • 고성능 PC가 push한 branch와 validation 결과
  • owner가 답해야 하는 질문
npm run guild-hall:night-watch:daily-work-packet -- --record

금지 동작

  • 금지고성능 PC가 main에 직접 push 또는 merge
  • 금지secret, token, credential, raw mail body 읽기
  • 금지allowed_write_paths 밖 수정
  • 금지AI가 만든 후보를 owner 승인 또는 auto-policy 승인 없이 ready로 실행
  • 금지validation 실패 branch를 승인 없이 push
