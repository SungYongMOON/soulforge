# 브라우저 검증 절차 (도구 비종속 — Codex/사람/어느 에이전트든)

verify_gate 의 기계 체크가 못 보는 화면 동작을 사람 또는 브라우저를 가진
에이전트가 수행한다. 소요 ~5분. 수행 결과는 packet `run_results` 또는
checklist `run_notes` 에 "BROWSER_QA: pass/fail + 발견" 한 줄로 남긴다.

전제: `node server.mjs` (또는 `node --watch server.mjs`) 가 떠 있고
http://127.0.0.1:4300 접속 가능. 콘솔(F12)을 열어 둔다.

1. 홈: KPI 4종 숫자 표시, 진행 과제 테이블 렌더, 미분류함 스트립 표시.
2. 과제 허브: 과제 행 클릭 → 개요/산출물/메일/이력 4탭 전환, 뒤로 ←홈.
3. 할일 쓰기: 허브 개요에서 제목 입력→추가→행 등장, [시작]→상태 변경.
4. 가이드: 작업 가이드 화면 → 산출물 카드 펼침 → 스텝 체크/해제 반영.
5. 메일: 메일 이력 → 행 체크 → 분류 바 등장 → (시연 시) 분류 실행 →
   대상 허브로 이동. 상세 패널 라벨 토글.
6. 모드 파리티: 표시 모드 전환(판타지) → 1~5 화면이 전부 사전 라벨로
   표시되는지(한쪽 모드 용어 잔존 = fail), 다시 업무 모드 복귀.
7. 콘솔: 위 전 과정에서 콘솔 에러 0 (경고는 기록만).
8. ⌘K 팔레트: 열림/검색/이동 1회.

fail 발견 시: 멈추지 말고 전 항목 마친 뒤, 발견 목록을 checklist
owner_questions 또는 버그 항목으로 기록하고 게이트는 FAIL 처리한다.

주의: 이 절차는 읽기/시연 쓰기만 한다. 실데이터 DB 에 시연 쓰기를 남겼으면
그 사실을 run_notes 에 명시한다 (무엇을, 어느 과제에).
