## summary
'입력이 충족되면 산출물 초안을 자동 생성'의 read-only 절반을 구현한다. 키스톤(ai_proposal) 의존이라 실제 쓰기는 막혀 있음 — (1) deliverable required input 을 artifact_requirement(scope_kind='deliverable_input')로 인코딩, (2) gateEval 의 boardCompleteness 충족판정 재사용한 read-only inputFulfillment(project_id), (3) /api/inputs/generate 는 충족돼도 자동 생성 0(409 미충족 또는 202 pending). 코어 LLM 0%·자동 쓰기 0. artifact_type 6종 사전 공유.

## data_model
- DDL/ALTER 없음 — artifact_requirement 재사용. 정책 행: scope_kind='deliverable_input', scope_key=<deliverable_name>, artifact_type=<필요 입력 키, 6종 사전>, label, mode='soft', seq. mode 기본 soft(미충족=버튼 비활성, 단계 hard 차단 아님).
- inputFulfillment 반환(계산만): [{deliverable_name,scope_key,required:[artifact_type...],satisfied:[...],missing:[{artifact_type,label}...],fulfilled:boolean}]. 충족 소스=boardCompleteness have Set 패턴(core_attachment.artifact_type 존재) 을 deliverable 단위로.
- 입력 증거=과제 링크 보드들의 첨부 합집합(기본값, boardCompleteness 재사용). gateEval 본문 변경 0(별 read-only 메서드).
