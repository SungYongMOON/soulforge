## data_model
- DDL 상수에 CREATE TABLE IF NOT EXISTS core_calculator ( id TEXT PRIMARY KEY, name TEXT NOT NULL, category TEXT, formula TEXT NOT NULL, variables TEXT, unit_out TEXT, source_ref TEXT, status TEXT NOT NULL DEFAULT 'draft', data_label TEXT NOT NULL DEFAULT 'real' );
- CREATE TABLE IF NOT EXISTS calculator_example ( id INTEGER PRIMARY KEY AUTOINCREMENT, calculator_id TEXT NOT NULL REFERENCES core_calculator(id), inputs_json TEXT NOT NULL, expected REAL NOT NULL, tolerance REAL NOT NULL DEFAULT 1e-6 );
- formula='Math.sqrt(a*a+b*b)' 화이트리스트 식(자작 파서 평가), variables=JSON [{name,label}], status='draft'|'active'(회귀 통과해야 active), source_ref=.registry/knowledge ref(선택). ALTER 불필요.

## code_changes
- store.mjs safeEval(formula,vars) 모듈 내부 순수 함수: (1) 토큰화(숫자/식별자/연산자 +-*/^()/쉼표). (2) 허용 식별자=변수명(vars 키)+허용 Math 멤버('sin','cos','tan','log','log10','sqrt','abs','pow','exp','min','max','PI','E'). 그 외 식별자/문자열/대괄호/세미콜론/할당 발견 시 throw 'unsafe_token'. (3) 재귀하강 파서 평가(eval/Function/new Function/vm 미사용 — 코드에 부재). NaN/Infinity 면 throw 'eval_error'.
- upsertCalculator({id,name,category,formula,variables,unit_out,source_ref,data_label}): name/formula 필수. formula 를 safeEval 더미 평가 시도 — unsafe_token throw 면 {error:'unsafe_formula'}(저장 거부). 통과 시 INSERT/UPDATE, status 초기 'draft'. 반환 {ok:true,id}.
- addCalculatorExample(calculator_id,inputs,expected,tolerance=1e-6): calculator 존재 가드. INSERT calculator_example(inputs_json,expected,tolerance). 반환 {ok:true}.
- evalCalculator(id,inputs): calculator 조회(없으면 error). safeEval(formula,inputs), throw 면 {error} 변환. 반환 {ok:true,value}.
- verifyCalculator(id): 모든 example 재계산 |value-expected|<=tolerance. 전부 통과 {ok:true,passed:N,failed:0}, 하나라도 실패 {ok:false,failures:[{example_id,got,expected}]}.
- activateCalculator(id): verifyCalculator ok 일 때만 UPDATE status='active'(회귀 강제). 실패면 {error:'examples_failed',failures}.
- server.mjs: GET /api/calculators→SELECT *; POST /api/calculators→upsertCalculator; POST /api/calculators/example→addCalculatorExample; POST /api/calculators/eval {id,inputs}→evalCalculator; POST /api/calculators/verify {id}→verifyCalculator; POST /api/calculators/activate {id}→activateCalculator+appendEvent({actor_kind:'human',kind:'calculator_activate',item_ref:id,used_refs:['calculator'],data_label:'real'}).
- lexicon.mjs 양 모드 동일 키: calc_title(계산기/계산 마법구), calc_formula(공식/주문), calc_run(계산/시전), calc_verify(검증 실행/검증), calc_activate(활성화/각인), calc_draft(초안/미각인), calc_active(활성/각인됨), calc_unsafe(허용되지 않은 식/금지된 주문), calc_example(검증 예제/검증 시료).
- app.js renderCalculators(): 목록(status 칩) + 변수 입력 폼→'계산'(POST /eval) + '검증 실행'(POST /verify pass/fail) + active 승격 버튼(검증 통과해야). 등록 폼(name/formula/variables), unsafe 거부 시 calc_unsafe.

## test_cases
- 'P-11: safeEval 화이트리스트 — 산술/Math 만': freshStore(); const id=store.upsertCalculator({name:'빗변',formula:'Math.sqrt(a*a+b*b)',variables:[{name:'a'},{name:'b'}]}).id; const r=store.evalCalculator(id,{a:3,b:4}); assert.equal(Math.round(r.value),5).
- 'P-11: 위험 식 거부 — process/대괄호/할당 차단': assert.equal(store.upsertCalculator({name:'x',formula:'process.exit(1)'}).error,'unsafe_formula'); assert.equal(store.upsertCalculator({name:'y',formula:'global["x"]'}).error,'unsafe_formula'); assert.equal(store.upsertCalculator({name:'z',formula:'a=1'}).error,'unsafe_formula').
- 'P-11: example 회귀검증 — 통과해야 active': freshStore(); const id=store.upsertCalculator({name:'합',formula:'a+b',variables:[{name:'a'},{name:'b'}]}).id; store.addCalculatorExample(id,{a:2,b:3},5); assert.ok(store.verifyCalculator(id).ok); assert.ok(store.activateCalculator(id).ok); store.addCalculatorExample(id,{a:1,b:1},99); const v=store.verifyCalculator(id); assert.equal(v.ok,false); assert.equal(store.activateCalculator(id).error,'examples_failed').
