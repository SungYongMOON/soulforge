## summary
만든 것·아는 것을 남기는 지식 카탈로그. core_knowledge(제목·요약·출처 포인터·키워드·.registry/knowledge ref) + core_attachment entity_type 화이트리스트에 'knowledge' 추가(포인터 첨부, 원문 미저장). 검색은 기존 _tokenize 재사용(retrieveKnowledge), FAQ+지식 통합(catalogSearch). RAG 포인터 인입=.registry/knowledge 정본을 source_ref 로 등재(파일 미파싱). FAQ(짧은 Q/A) vs knowledge(출처기반 카드) 분리.

## data_model
- DDL 상수에 CREATE TABLE IF NOT EXISTS core_knowledge ( id TEXT PRIMARY KEY, title TEXT NOT NULL, summary TEXT, topic TEXT, source_ref TEXT, claim_ceiling TEXT DEFAULT 'source_supported', keywords TEXT, pointer TEXT, data_label TEXT NOT NULL DEFAULT 'real' );
- core_attachment entity_type 검증 배열(store.mjs L1210)을 ['item','project','purchase','meeting','part'] → ['item','project','purchase','meeting','part','knowledge']. 스키마 변경 0(코드 화이트리스트만).
- source_ref='.registry/knowledge/<id>' 포인터(미파싱), claim_ceiling 출처 등급, pointer 외부 원문 위치(원문 미저장), keywords 매칭어.

## code_changes
- store.mjs upsertKnowledge({id,title,summary,topic,source_ref,claim_ceiling,keywords,pointer,data_label}): title 필수(없으면 {error:'title_required'}). id=id||`kn_${randomBytes(4).toString('hex')}`. INSERT ... ON CONFLICT(id) DO UPDATE. 반환 {ok:true,id}.
- knowledge({topic=null,q=null}={}): SELECT * + topic/q(title/summary/keywords LIKE) 필터 ORDER BY title LIMIT 300.
- retrieveKnowledge(question,n=5): this._tokenize 재사용. core_knowledge title+summary+keywords+topic hay 점수화(완전일치 1·부분일치 0.6, retrieveFaqMany 규칙). 상위 n {knowledge,score,norm}. 추론 아님.
- catalogSearch(question,n=5): retrieveFaqMany(question,n)와 retrieveKnowledge(question,n)에 type('faq'|'knowledge') 부착해 score DESC 병합. 외부전송 0·추론 0.
- addAttachment entity_type 검증 배열에 'knowledge' 추가.
- server.mjs: GET /api/knowledge {topic,q}→store.knowledge(...); POST /api/knowledge→store.upsertKnowledge({...body,data_label:'real'})+appendEvent({actor_kind:'human',kind:'knowledge_upsert',item_ref:r.id,to:body.title,used_refs:body.source_ref?[body.source_ref]:[],data_label:'real'}); GET /api/knowledge/search {q}→store.catalogSearch(qp.q). .registry/knowledge 미read.
- lexicon.mjs 양 모드 동일 키: kn_catalog_title(지식 카탈로그/대도서관), kn_add(지식 등재/전승 봉헌), kn_source(출처/전거), kn_claim_ceiling(근거 등급/전승 등급), kn_search_ph(규칙·규격·매뉴얼 검색/전승 검색), kn_no_result(관련 지식 없음/전승 없음), kn_type_faq(매뉴얼/법전), kn_type_knowledge(지식 카드/전승 카드).
- app.js renderKnowledge() 확장(L375 기존): 상단 검색창(api('/api/knowledge/search?q=')) + 결과 type 칩(kn_type_*)+source_ref 표시. 하단 '지식 등재' 폼(POST /api/knowledge). source_ref 는 클릭 불가 텍스트(정본 포인터).

## test_cases
- 'P-10: 지식 등재 + _tokenize 재사용 검색': freshStore(); store.upsertKnowledge({title:'케이블 라벨 규칙',summary:'라벨은 양끝에',topic:'wiring',keywords:'케이블,라벨',source_ref:'.registry/knowledge/source_criticism'}); const hits=store.retrieveKnowledge('케이블 라벨'); assert.ok(hits.length>=1); assert.ok(hits[0].knowledge.title.includes('케이블')); assert.equal(store.knowledge({topic:'wiring'}).length,1).
- 'P-10: catalogSearch FAQ+지식 통합(type)': freshStore(); store.upsertFaq({question:'재고 부족 처리',answer:'발주 요청',keywords:'재고,부족'}); store.upsertKnowledge({title:'재고 안전기준',summary:'min_qty 기준',keywords:'재고,안전'}); const r=store.catalogSearch('재고'); assert.ok(r.some(x=>x.type==='faq')); assert.ok(r.some(x=>x.type==='knowledge')).
- 'P-10: knowledge 엔터티 첨부 포인터(원문 미저장)': freshStore(); const id=store.upsertKnowledge({title:'규격서'}).id; const a=store.addAttachment({entity_type:'knowledge',entity_id:id,name:'spec.pdf',pointer:'/proto/spec.pdf'}); assert.ok(a.ok); assert.equal(store.attachments({entity_type:'knowledge',entity_id:id}).length,1); assert.equal(store.addAttachment({entity_type:'badtype',entity_id:'x',name:'n',pointer:'p'}).error,'bad_entity_type').
