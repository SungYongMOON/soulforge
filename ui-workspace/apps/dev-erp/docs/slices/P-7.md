## summary
두 읽기 전용 롤업을 새 테이블 0으로 추가. 사람별 부하=assignee_ref 별 미완/차단/연체 GROUP BY. 회의 미결=meeting_action_map 의 미완 액션 회의별 롤업. 둘 다 집계만(저장 0, 개인 점수 0). P-6 의 사람·지원 축·lexicon 패턴 재사용이라 depends_on=P-6.

## data_model
- 신규 테이블/컬럼 0. core_item(assignee_ref,status,due), core_meeting(id,title,project_id), meeting_action_map(meeting_id,item_id) 만.
- 부하 집계: assignee_ref 별 open_cnt/blocked_cnt/overdue_cnt(due<today AND status!='done'). NULL=‘(미배정)’.
- 회의 미결: meeting_action_map JOIN core_item, 회의별 total_actions/open_actions(status!='done'). open_actions>0 만.

## code_changes
- store.mjs workload(todayKey): SELECT assignee_ref, COUNT(*) total, SUM(status!='done') open_cnt, SUM(status='blocked') blocked_cnt, SUM(status NOT IN ('done') AND due IS NOT NULL AND due<?) overdue_cnt FROM core_item GROUP BY assignee_ref. people() 로 이름 매핑(매칭 없으면 ref, NULL→'(미배정)'). open_cnt DESC. 점수 미산출.
- store.mjs meetingOpenRollup(): SELECT m.id,m.title,m.project_id, COUNT(map.item_id) total_actions, SUM(CASE WHEN i.status!='done' THEN 1 ELSE 0 END) open_actions FROM core_meeting m JOIN meeting_action_map map ON map.meeting_id=m.id JOIN core_item i ON i.id=map.item_id GROUP BY m.id HAVING open_actions>0 ORDER BY open_actions DESC.
- server.mjs: GET /api/workload→store.workload(new Date().toISOString().slice(0,10)); GET /api/meetings/open→store.meetingOpenRollup(). 둘 다 GET, event_log 미기록.
- lexicon.mjs 양 모드 동일 키: workload_title(사람별 부하/유닛별 적재), workload_unassigned((미배정)/(미배정)), col_open(미완/미처치), col_blocked(차단/봉인), col_overdue(연체/기한초과), meeting_open_title(회의 미결/원탁 미결), col_open_actions(미결 결의/미결 결의).
- app.js widgetBody: id==='workload'(api('/api/workload') 사람별 표 + cap_support_note 재사용), id==='meeting_open'(api('/api/meetings/open') 회의별 미결 표).
