## summary
사람을 Soulforge 역량 캐논에 ref 문자열로 링크해 '누가 무엇을 잘하나'를 데이터로 만든다. core_person 에 unit_ref/capability_label 2컬럼(ALTER 멱등), person_skill 매핑(N:N). .registry/.unit 은 read-only — YAML 파싱 0, ref 문자열만. 콕핏 nudges='먼저 할 것'(연체>차단>오늘 우선순위로 1인당 상위 할일, 읽기 전용·점수 미저장). 감시 아닌 지원.

## data_model
- openStore for(ddl) try/catch 블록(L360-374)에 ALTER 2줄: 'ALTER TABLE core_person ADD COLUMN unit_ref TEXT', 'ALTER TABLE core_person ADD COLUMN capability_label TEXT'.
- DDL 상수에 CREATE TABLE IF NOT EXISTS person_skill ( person_id TEXT NOT NULL REFERENCES core_person(id), capability_label TEXT NOT NULL, source_ref TEXT, weight REAL NOT NULL DEFAULT 1, PRIMARY KEY (person_id, capability_label) );
- 개인 평가점수 컬럼 만들지 않음(감시경계). unit_ref='.unit/<id>' 포인터(미파싱), source_ref='.registry/skills/<id>' 또는 classes ref(미파싱), weight=정렬용(점수 아님).

## code_changes
- store.mjs upsertPerson 확장: 기존 시그니처 보존하되 unit_ref/capability_label 도 받아 ON CONFLICT 업데이트에 COALESCE(미전달 시 기존값 유지). upsertPerson({id,name,role,unit_ref,capability_label,data_label}).
- setPersonSkill(person_id,capability_label,{source_ref=null,weight=1}={}): core_person 존재 가드(없으면 {error:'person_not_found'}). INSERT ... ON CONFLICT(person_id,capability_label) DO UPDATE SET source_ref,weight. 반환 {ok:true}.
- personSkills(person_id): SELECT capability_label,source_ref,weight WHERE person_id=? ORDER BY weight DESC.
- capabilityMatrix(): people() 전체에 personSkills 묶어 [{person_id,name,role,unit_ref,capability_label,skills:[...]}]. 개인 점수 산출/저장 0.
- nudges({person=null,limit=5}={}): 읽기 전용. SELECT core_item WHERE status NOT IN ('done') AND (assignee_ref=? OR ?=''). today=new Date().toISOString().slice(0,10). 우선순위 overdue(due<today)>blocked>due_today>open, 각 항목 reason 라벨 부착. 점수 미저장(쿼리시 계산). 반환 상위 limit [{id,title,project_id,due,status,assignee_ref,reason}].
- server.mjs: GET /api/capability/matrix→store.capabilityMatrix(); GET /api/people 유지(people() SELECT * 라 신규 컬럼 자동 포함); POST /api/people/skill {person_id,capability_label,source_ref,weight}→store.setPersonSkill(...)+appendEvent({actor_kind:'human',kind:'person_skill_set',item_ref:person_id,to:capability_label,used_refs:source_ref?[source_ref]:[],data_label:'real'}); GET /api/nudges→store.nudges({person:qp.person,limit:qp.limit?Number(qp.limit):5}). .unit/.registry 파일 미read.
- lexicon.mjs 양 모드 동일 키: nav_team(사람·역량/길드원), cap_matrix_title(능력 매트릭스/역량표), nudge_title(먼저 할 것/우선 토벌), nudge_reason_overdue(연체/기한초과), nudge_reason_blocked(차단/봉인), nudge_reason_due_today(오늘 마감/금일 마감), nudge_reason_open(진행 중/교전 중), cap_support_note(개인 점수 없음 · 평가 아닌 지원/등급 없음 · 평가 아닌 지원).
- app.js: widgetBody(id) 에 id==='nudges' 케이스(const ns=await api('/api/nudges'); 상위 5 miniRow([title,project_id,L['nudge_reason_'+reason]]); 제목 L.nudge_title). renderCapability() 추가(api('/api/capability/matrix') 사람×능력 칩 표 + 하단 L.cap_support_note 고지). render() switch 에 연결(group_team 진입, 새 nav 모듈 미등록=기존 group 재사용). nudges 위젯은 WIDGET_PLAN 에 추가하지 않고 widgetBody 분기만(ready 위젯 자동 카탈로그 정합 위해 별도 등록은 owner 요청 시).
