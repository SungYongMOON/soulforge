# B9 — 줄기 강(江) 뷰: 맥락 그래프의 목표 모양 (owner 비전 정본)

- 상태: 설계 정본 (2026-07-06 owner 구술 → claude_fable-5 정리). 구현 전.
- 상위 정본: `STEM-V2-ONTOLOGY.md` (골격/작업/이력 3종, 연결 등급, 드래그=사람 확정 — 전부 그대로 유효. 본 패킷은 온톨로지가 아니라 **읽는 모양**의 정본)
- 관계: B8(지도 v2, 방사형)은 "무엇이 얼마나 쌓였나" 조감 입구로 유지. 맥락(이야기)을 읽는 주 표면은 본 패킷의 뷰가 맡는다.

## 1. Owner가 원하는 것 (2026-07-06 구술 요지)

- 시간순 나열이 아니라 **한 목적(맥락)이 한 줄** — "한번 일한 게 점이면, 그게 한 맥락에서는 한 줄". GitHub 커밋 그래프·마인드맵에 가까운 모양.
- 기둥 = 체계공학 흐름(큰 강줄기). **PDR/CDR 게이트는 기둥 위의 큰 점(이정표)**. 문서 작성·설계·설계리뷰는 전부 "그 큰 줄기를 위해 한 일"로 기둥/가지 아래에 모여야 한다.
- **잎을 하나 잡으면**: 어디서부터 온 건지(기원)·어떻게 끝난 건지(종결)를 바로 조회. 그 잎과 연결된 줄기만 **잘라서** 볼 수도 있어야 한다.
- **나무의 모양 자체가 진단**: 했지만 다음에 안 쓴 가지(죽은 가지), 가지만 많음(분산/쓸데없는 일처럼 보임), 가지 희소(집중 또는 일 안 함 — 해석 병기), 언제 일이 몰렸나, 누가 많이 해결했나, 요청이 어떤 식으로·언제 오는가.
- 최종 물음에 몇 초 안에 답: "이거 누가 언제 왜 시켰더라?"

## 2. 나무의 문법 (렌더·데이터 공통 계약)

| 구성 | 뜻 | 데이터 (전부 기존 — 새 수집 없음) |
|---|---|---|
| 기둥 | 과제 SE 흐름, **가로축 = 시간** | skeleton 줄기 + 게이트 일정 |
| 큰 점 | 게이트/마일스톤 (PDR·CDR) | gates |
| 가지 | 하나의 목적/맥락 (작업줄기·이력줄기) | `branches.csv` (born_at/closed_at/anchor_ref) |
| 점 | 한 번의 일: 메일 수신(←)/발신(→), 할일 전환·완료, 산출물 등록 | core_mail 방향·일시, event_log, deliverables |
| 잎 | 개별 기록 1건 | 각 원장 행 |

- 가지는 기둥(또는 상위 가지)의 **시간 좌표에서 갈라져 나와** 자라고, 닫힘점에서 끝난다. 방사형 아님.
- 이력줄기(반복 회의)는 기둥과 나란히 달리는 점선 레일, 회차가 점 (`occurrences.csv`).
- 확정 대기 제안은 점선 유지(자동확정 금지 — 온톨로지 원칙).

## 3. 잎 추적 계약 (leaf → story)

잎/가지 클릭 시 3단 즉답:
1. **기원** — 이 가지의 첫 점: 누가(발신자)·언제·무슨 요청(제목). 스레드 첫 수신 메일 또는 할일 생성 이벤트.
2. **경로** — 가지 위 점들 시간순, 방향 표기(←받음/→보냄), 첨부 수·담당 변화 포함.
3. **종결** — 완료 일시·산출물·제출(발신) 기록. 미종결이면 "열려 있음 + 마지막 움직임 날짜".

필요 조인: 이력키(스레드 묶음), item `spawned_from`/`origin_occurrence_ref`, completion_log, deliverables, `sources.csv.branch_ref`. **전부 기존 필드.** "그 잎과 연결된 줄기만 잘라 보기" = 해당 branch_ref 필터 뷰.

## 4. 모양 진단 뷰 (파생 통계, 읽기전용)

- 죽은 가지: 닫힌 뒤 참조 0·지식 승격 0 → 회색 처리 (다음에 쓰인 가지와 시각 구분)
- 몰림 히트맵: 주 단위 점 밀도 띠(기둥 아래)
- 사람별 분포: 가지/점의 담당·해결 집계
- 요청 패턴: 방향=수신 점의 발신자·시기 분포
전부 원장 무변경 파생. 뷰가 곧 해석을 강요하지 않게 "집중/정체" 같은 이중 해석 라벨은 병기.

## 5. 지식그래프와의 경계 (owner 질문 답)

- **맥락 그래프 = 일이 어떻게 흘렀나** (시간·사람·요청·방향). **지식 그래프 = 무엇을 알게 됐나** (주제·자료·근거 — "모터 드라이브" 검색은 이쪽). 목적 분리 유지.
- 접점은 잎 수준 양방향: ① 일의 종결 → 지식 후보 (**이미 배선**: E2 completion feed) ② 지식 검색 결과 → "이 자료가 태어난 가지"로 점프 (`sources.csv.branch_ref` 역링크 재사용, 신규 수집 없음).
