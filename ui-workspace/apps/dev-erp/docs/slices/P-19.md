## summary
P-4-ai 의 createProposal/approveProposal 단일 표면 위에 결정적 규칙 스캐너들이 '추천'을 생성하는 어댑터를 얹는다. 각 추천기는 도메인을 읽어 갭/이상을 찾고 createProposal(pending)로만 적재 — 자동 쓰기 0, 모든 적용은 사람 approve. MVP 추천기 3종만 구현, 나머지는 후속 슬라이스 자리만 주석(스캐너 1=슬라이스 1). 추천 근거는 결정적 규칙(LLM 아님). P-4-ai 키스톤 미정이면 전체 막힘.

## data_model

- 추천기↔PROPOSAL_KINDS 매핑: (a) 산출물 갭→kind='create_item', (b) 필수 기술자료 요구 누락→kind='set_artifact_requirement', (c) 부품-과제 링크 누락→kind='link_part_project'. 새 kind 필요 시 P-4-ai 화이트리스트에 먼저 추가(슬라이스 분리).
