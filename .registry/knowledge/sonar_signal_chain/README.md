# sonar_signal_chain

`sonar_signal_chain` is a public-safe registry knowledge entry for sonar engineering orientation.

It registers the reusable sourcebound map from underwater acoustics and hydrophone sensing through AFE, ADC, digital front-end processing, beamforming, detection, and calibration.

## Boundary

- This entry is routing/orientation knowledge, not a complete sonar design handbook.
- It does not approve a production hardware design.
- It does not include NotebookLM answers, raw source payloads, private projections, or weak web-source claims.
- It keeps CFAR/ROC/Pd/Pfa as an explicit source-gap lane until stronger detection-theory sources are packetized.

## Source Posture

The source list in `knowledge.yaml` uses public official/vendor/standards-style references. Detailed source packet evidence and NotebookLM advisory metadata remain in `_workmeta/system` rather than this public registry entry.
