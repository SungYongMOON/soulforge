# Report Writer — 빈칸 채우기 인터뷰 (grill 형식)

목적: 팀원이 측정 자료나 초안만 들고 왔을 때, 결론을 만드는 빠진 조각을 **질문으로 끌어내** 채운다.
`grill_me` 처럼 한 번에 한 질문, 답을 받을 때마다 골격을 갱신한다. 사실/수치는 지어내지 않는다 —
팀원에게서 받거나 "미확인"으로 둔다.

## 언제 인터뷰 모드인가

- 입력이 측정 자료/그래프/초안뿐이고 결론이 없을 때 (데이터 덤프)
- So-What ①왜 / ④그래서 / ⑤다음 중 빠진 게 있을 때
- 종합 판정, 측정 불확도, 적용 규격 ID, 권고가 비어 있을 때

정보가 이미 충분하면 인터뷰를 생략하고 바로 초안으로 간다.

## 절차

1. 대상 식별: 무슨 보고/발표이고 어떤 자료가 있는지 한 질문으로 확인.
2. 갭 스캔: So-What 5문 + 필수 필드(종합 판정/불확도/규격 ID/권고)를 입력에 매핑해 빈칸 목록을 만든다.
3. 우선순위: **①왜 → ④그래서 → ⑤다음** 순으로(가장 자주 비고 가장 중요), 그다음 종합 판정·불확도·권고.
4. **한 번에 한 질문.** 각 질문에 "왜 묻는지" 한 줄 + 보기/예시를 붙인다.
5. 답을 받을 때마다 골격을 갱신해 보여준다.
6. 모르는 답은 강요하지 않고 "미확인(확정 조건: ~)"으로 표시한다.
7. ①④⑤가 채워지면 타입 템플릿으로 확장 → 별도 de-slop 패스 → 체크리스트로 닫는다.

## 질문 뱅크 (빈칸별)

| 빈칸 | 질문 (예시) |
| --- | --- |
| ① 왜 | "이 시험/작업을 안 했으면 무엇을 모르거나 못 정했을까요? 이번 결과로 내려야 할 결정이 뭐죠?" |
| ② 뭘 | "대상·조건·횟수를 한 줄로요. (예: 시제 N개, 75 kHz, 각 5회)" |
| ③ 뭘얻 | "핵심 수치 1~2개만, 단위 포함으로요. (그래프 말고 숫자로)" |
| ④ 그래서 | "그 수치가 기준 대비 합격인가요 불합격인가요? 종합으로 한 문장이면?" |
| ⑤ 다음 | "그래서 뭘 하시겠어요? 상대에게 무엇을 결정/승인해 달라는 거죠?" |
| 종합 판정 | "어느 규격/기준 문서 대비 판정인가요? (문서 ID)" |
| 불확도 | "측정 불확도나 신뢰수준(k) 아세요? 모르면 미확인으로 둘게요." |
| 권고→의사결정 | "권고가 받아들여지면 누가 무엇을 결정하나요? 미채택 시 영향은요?" |

## 경계

- 사실·수치·판정을 대신 만들지 않는다. 답이 없으면 미확인 또는 owner question.
- 한 번에 한 질문. 답마다 골격 갱신. 팀원이 "이제 작성"이라고 하면 초안을 만든다.
