# .registry/docs/architecture

## 목적

- `.registry` 의 owner-local 구조 설명을 두는 active 문서 진입점이다.
- species inline heroes, classes entry rule, reserved canon buckets 를 이 경로에서 설명한다.
- skill canon 과 executor bridge 경계는 `SKILL_CANON_BOUNDARY.md` 에서 설명한다.

## 현재 상태

- Active seeded canon surface
- species, class, skill, tool, knowledge owner 경계를 이 경로에서 설명한다.
- sample schema 와 example YAML 세트는 필요한 범위만 점진적으로 추가한다.
- active boundary note:
  - [`SKILL_CANON_BOUNDARY.md`](SKILL_CANON_BOUNDARY.md)
