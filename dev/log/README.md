# dev/log

## 목적

- `log/` 는 Soulforge 개발 과정의 실제 수행 결과를 기록한다.
- 계획이 아니라 수행 후 기록을 모은다.

## 포함 대상

- 날짜별 작업 로그
- 변경 이유, 영향, 남은 리스크
- `YYYY-MM-DD_<topic>_log.md` 형식의 기록 파일

## 제외 대상

- 수행 전 계획
- 저장소 전체 구조 계약 문서와 프로젝트 전용 로그

## 관련 경로

 - [dev/README.md](../README.md)
 - [dev/plan/README.md](../plan/README.md)

## 상태

- Stable
- 저장소 공용 개발 이력 문서를 유지한다.
- project 전용 로그는 각 프로젝트의 `.project_agent/` 아래에 둔다.
- 최신 로그에는 `.agent` 최종 구조 정합화와 root 문서 정리를 포함한다.
